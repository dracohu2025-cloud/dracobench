from collections import OrderedDict

def solve(capacity: int, ops: list[str]) -> list[int]:
    result = []
    cache = OrderedDict()
    
    for op in ops:
        parts = op.split()
        if parts[0] == "put":
            if capacity == 0:
                continue
            key = int(parts[1])
            value = int(parts[2])
            if key in cache:
                cache.move_to_end(key)
                cache[key] = value
            else:
                if len(cache) >= capacity:
                    cache.popitem(last=False)
                cache[key] = value
        elif parts[0] == "get":
            if capacity == 0:
                result.append(-1)
                continue
            key = int(parts[1])
            if key in cache:
                cache.move_to_end(key)
                result.append(cache[key])
            else:
                result.append(-1)
    
    return result

## Bug 分析

在 Python 中，若 `finally` 子句里也执行 `return`，该返回值会**覆盖** `try`/`except` 中的返回值。因此 `finally: return None` 把 `try` 块里的 `42` 抹掉了，函数始终返回 `None`。

这是 PEP 255 明确规定的语义：异常/正常返回都会被 `finally` 的 `return` 拦截并替换。

## 最小修复

```python
def read_value():
    try:
        return 42
    finally:
        pass  # 或仅做清理工作，不要 return
```

如果 `finally` 块只是为了存在而存在，可以直接删掉整个 `finally`：

```python
def read_value():
    return 42
```

**原则**：`finally` 只用于释放资源、关闭文件、恢复状态等**副作用操作**，不要在其中 `return`（也不要在其中 `break`/`continue` 进入外层循环）。

def solve(expr: str) -> int:

    n = len(expr)

    def skip_spaces():
        nonlocal pos
        while pos < n and expr[pos] == ' ':
